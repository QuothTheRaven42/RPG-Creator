# RPG Creator

`RPG Creator` is a small command-line Python project for building RPG characters and running a simple party-vs-enemy battle. It combines random character generation, race and class bonuses, inventory management, combat, leveling, and character-sheet export into a single interactive script.

## What It Does

- Creates a party of `1` to `6` player characters.
- Supports `Barbarian`, `Cleric`, `Wizard`, `Sorcerer`, `Fighter`, and `Rogue`.
- Supports `Human`, `Dwarf`, `Elf`, `Gnome`, and `Halfling`.
- Randomizes starting HP and core stats, then applies race and class modifiers.
- Runs a turn-based battle against one selected enemy type and enemy count.
- Tracks inventory, item usage, damage, experience, and level-ups.
- Exports finished character sheets to `.txt` files.

## Project Files

| File | Purpose |
| --- | --- |
| `main.py` | Interactive entry point and battle loop |
| `classes.py` | Playable class definitions and class-specific bonuses |
| `character.py` | Base player-character behavior, stats, inventory, and export logic |
| `combatant.py` | Shared combat behavior for players and enemies |
| `enemies.py` | Enemy classes and enemy factory helpers |
| `*_lvl1.txt` | Example exported character sheets generated by the app |

Folders such as `.idea`, `.venv`, `.mypy_cache`, and `__pycache__` are development/runtime artifacts and are not required for understanding the game logic.

## Requirements

- Python `3.10+`
- No third-party dependencies

## Run

From the project folder:

```powershell
python main.py
```

## Gameplay Flow

When you run `main.py`, the script will:

1. Ask how many players are in the party.
2. Ask which class each player should use.
3. Prompt for each character's name and race.
4. Print each generated character sheet immediately after creation.
5. Ask how many enemies are present and which enemy type to spawn.
6. Run the battle loop until the party or enemy group is defeated.
7. Offer to export all character sheets at the end of the fight.

## Playable Classes

| Class | Hit Dice | Adjustments |
| --- | --- | --- |
| Barbarian | `20` | `+3 STR`, `+3 CON`, `-3 INT` |
| Cleric | `6` | `+3 CON`, `+3 WIS`, `-3 CHA` |
| Wizard | `8` | `+3 INT`, `+3 CHA`, `-3 max HP` |
| Sorcerer | `16` | `+3 WIS`, `+3 max HP`, `-3 DEX` |
| Fighter | `14` | `+3 STR`, `+3 CHA`, `-3 WIS` |
| Rogue | `8` | `+3 DEX`, `+3 INT`, `-3 CON` |

## Playable Races

| Race | Adjustments |
| --- | --- |
| Human | `+1 STR`, `+1 CON`, `+1 INT`, `+1 WIS`, `+1 CHA` |
| Dwarf | `+2 STR`, `+2 CON`, `+1 WIS` |
| Elf | `+2 DEX`, `+1 INT`, `+1 WIS`, `+1 CHA` |
| Gnome | `+2 DEX`, `+2 INT`, `+1 CON` |
| Halfling | `+2 DEX`, `+1 CON`, `+2 CHA` |

## Enemy Types

| Enemy | HP | Attack Die |
| --- | --- | --- |
| Goblin | `10` | `d5` |
| Skeleton | `15` | `d10` |
| Dragon | `100` | `d30` |

The current battle script spawns one enemy instance at a time and uses a counter to represent how many of that enemy remain.

## Creating Characters in Code

The interactive game uses `from_prompt()`, but you can also create characters directly:

```python
from classes import Barbarian, Wizard

barbara = Barbarian("Barbara", "Dwarf")
david = Wizard("David", "Elf")
```

Or prompt interactively for a specific class:

```python
from classes import Rogue

player = Rogue.from_prompt()
```

## Character Output

Exported files follow this naming pattern:

```text
Name_the_Race_class_lvl1.txt
```

Example:

```text
Barbara_the_Dwarf_barbarian_lvl1.txt
```

Each exported file contains the character sheet followed by the inventory list.

## Current Limitations

- `import_character()` exists in `main.py` but is marked unfinished and is not used by the game loop.
- The current CLI battle only supports one enemy type per encounter.
- There are no automated tests in the repository.
